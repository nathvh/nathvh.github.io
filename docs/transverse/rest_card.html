    En résumé

    • Representational State Transfer
    • style d'architecture par services web
    • utilisation du protocole HTTP /HTTPS
      • codes retours
      • informations dans body/header
      • surtout GET, POST, PUT et DELETE
      • mais aussi HEAD, PATCH, CONNECT, OPTIONS et TRACE
    • interopérabilité des systèmes
    • pas d'état : stateless
    • mise en cache

    HTTP

    Méthodes

    Méthode Utilisation
    GET Récupération de ressource, n'a aucun impact sur cette dernière
    POST Envoi de données afin de gérer, modifier ou ajouter des données
    PUT Remplacement ou ajout d'une ressource
    DELETE Suppression d'une ressource du serveur
    PATCH Modification partielle d'une ressource
    HEAD Récupération d'information sur la ressource sans demander la ressource
    OPTIONS Récupération des options de communication
    CONNECT Permet l'usage d'un proxy
    TRACE Le serveur doit retourner les données reçues, pour test ou diagnostic

    URI & exemples

    Méthodes URI Description
    POST api/fiches crée une fiche grâce aux données dans le corps de la requête. Le retour doit fournir l'id de la nouvelle fiche
    GET api/fiches récupère toutes les fiches Devrait être paginée
    GET api/fiches/{uid} récupère une seule fiche dont l'identifiant unique est {uid}
    PUT api/fiches/{uid} modifie la fiche avec l'identifiant unique {uid} dont les informations sont dans le corps de la requête
    DELETE api/fiches/{uid} supprime la fiche avec l'identifiant unique {uid}

    Hiérarchisation

    URI Description
    http://mywebsite.com/fiches toutes les fiches
    http://mywebsite.com/fiches/42 la fiche 42
    http://mywebsite.com/fiches/42/versions toutes les versions de la fiche 42
    http://mywebsite.com/fiches/42/versions/8 la version 8 de la fiche 42

    Code retour

    Code d'état

    Code d'état État
    1xx Informations
    2xx Succès
    3xx Redirection
    4xx Erreur client
    5xx Erreur serveur

    Les basiques

    Code Message Description
    200 OK Le code par défaut en cas de succès
    201 Created La ressource a été créée avec succès
    204 No-content L'action est réussie mais le corps retour est vide
    401 Unauthorized Authentification nécessaire
    403 Forbidden Authentification et droits nécessaires
    404 Not Found La ressource ou l’URI n'existe pas
    500 Internal Server Error Erreur générique côté serveur
    504 Gateway Time-out Le serveur n'a pas répondu dans le temps imparti
